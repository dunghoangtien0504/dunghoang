Setup OpenAI + Facebook API
BƯỚC 01
Đăng ký OpenAI API key
Một bài đăng Facebook hoàn chỉnh = ảnh + văn bản (caption). Hôm nay agent của bạn sẽ tự sinh ra cả 2 phần trong cùng 1 lần. Phần ảnh dùng GPT Image API của OpenAI (model gen ảnh tốt nhất hiện tại, text trong ảnh viết chuẩn tiếng Việt, giá rẻ). Phần văn bản dùng GPT-4 hoặc Claude (qua chính agent coding của bạn).

1.1 — Tạo tài khoản OpenAI Platform
Vào https://platform.openai.com (chú ý platform, không phải chatgpt.com).
Đăng ký tài khoản mới (hoặc đăng nhập nếu đã có).
Tài khoản mới được tặng $5 free credit — đủ gen ~500-1000 ảnh chất lượng thường (đủ cho ~1 năm content nếu đăng mỗi ngày).
1.2 — Tạo API key
Settings (icon ⚙️ góc trên phải) → API keys → Create new secret key → đặt tên dh-content-bot → bấm Create
Một popup hiện ra với chuỗi key bắt đầu bằng sk-proj-.... Copy ngay và lưu vào file Notepad/notes — popup đóng là không xem lại được, phải tạo key mới.

Kiểm tra điểm
Bạn có 1 chuỗi bắt đầu bằng sk-proj-, dài khoảng 100-150 ký tự. Đây là OpenAI API key của bạn.

1.3 — Nạp tiền (nếu hết $5 free)
$5 free đủ làm Ngày 17. Nhưng để dùng lâu dài, có 2 cách nạp:

Visa/Mastercard quốc tế (Vietcombank Visa Debit, Techcombank Mastercard...): vào Settings → Billing → Add payment method → nạp tối thiểu $5.
Wise / Revolut: nếu thẻ ngân hàng VN bị từ chối (hay xảy ra với account mới), dùng thẻ ảo Wise — nạp VND vào Wise → tạo thẻ ảo USD → add vào OpenAI.
AI-First — kẹt là hỏi agent
Nếu màn hình OpenAI khác hướng dẫn (UI thay đổi liên tục), screenshot màn hình → paste vào agent coding + bảo: "Tôi đang ở đây trên OpenAI Platform, muốn tạo API key, bấm tiếp đâu?". Agent đọc ảnh và chỉ chính xác.

⚠️ Troubleshoot — Bước 01
"Account của tôi bị khoá ngay sau khi đăng ký"
"Thẻ Visa của tôi bị từ chối"
"Tôi không thấy $5 free credit"
BƯỚC 02
Tạo Facebook Developer App
Để code Python đăng bài lên Page Facebook, ta cần 1 Facebook Developer App. App này là "chìa khóa" giúp Facebook nhận biết code của bạn.

Hiểu kiểu đơn giản
Facebook Developer App giống như "thẻ nhân viên" bạn cấp cho code của mình. Có thẻ này, code của bạn được phép vào "tòa nhà Facebook" và làm việc thay bạn — như đăng bài lên Page bạn quản lý.
2.1 — Tạo App tại Facebook for Developers
Vào https://developers.facebook.com — đăng nhập bằng tài khoản Facebook cá nhân của bạn (tài khoản mà bạn đang admin Page kinh doanh).
Nếu chưa từng vào, Facebook sẽ yêu cầu bạn "Register as developer" — đồng ý điều khoản.
Bấm góc trên phải My Apps → Create App.
2.2 — Chọn loại App
Facebook hỏi "What do you want your app to do?". Chọn:

Other → bấm Next → chọn Business → bấm Next
Quan trọng
Phải chọn loại Business. Loại "Consumer" hoặc "Gaming" không có Marketing API. Sai loại = phải tạo lại app từ đầu.

2.3 — Đặt tên + tạo
App name: dh-content-bot (hoặc gì cũng được, không có chữ "facebook" hay "fb").
Email: email của bạn.
Business Account: chọn Business Manager nếu có; nếu không, để trống cũng được.
Bấm Create App — Facebook hỏi password, nhập password Facebook cá nhân.
Kiểm tra điểm
Bạn được chuyển vào trang Dashboard của App, thấy tên app + một ID dạng 1234567890123456 (15-16 chữ số) ở phía trên. Đây là App ID — sẽ cần ở Bước 03.

2.4 — Lấy App Secret
Sidebar trái → App settings → Basic → tìm dòng App Secret → bấm Show → nhập password Facebook → copy chuỗi
App Secret là chuỗi 32 ký tự. Lưu vào notes cùng với App ID.

Bí mật tuyệt đối
App Secret giống như mật khẩu — KHÔNG share, KHÔNG commit lên Git. Sẽ lưu vào file .env ở Bước 05.

AI-First — UI Facebook đổi liên tục
Facebook for Developers cập nhật giao diện rất thường xuyên. Nếu bạn không thấy nút giống mô tả, screenshot màn hình → paste agent coding + hỏi: "Tôi đang ở Facebook Developer App, muốn tạo App Business / lấy App Secret, bấm đâu?". Đừng đoán mò.

⚠️ Troubleshoot — Bước 02
"Bị block không cho tạo App"
"Không thấy option Business khi tạo App"
"Quên App ID/Secret rồi"
BƯỚC 03
Lấy Permanent Page Access Token
Đây là phần khó nhất ngày hôm nay. Đọc kỹ, làm chậm. Mục tiêu cuối: có 1 chuỗi token không bao giờ hết hạn, dùng cho code đăng bài lên Page mãi mãi.

AI-First — luật vàng cho Bước 03
Bước này có nhiều thuật ngữ kỹ thuật (token, OAuth, curl, JSON...). Đừng cố hiểu hết — chỉ cần làm đúng thứ tự. Mỗi sub-step:

Đọc hướng dẫn 1 lượt
Có chỗ không hiểu → screenshot + hỏi agent coding "giải thích chỗ này"
Có lệnh curl → paste nguyên lệnh vào agent + bảo "chạy giúp tôi, in output ra"
Lỗi → copy error → paste agent + bảo "lỗi này nghĩa là gì, sửa sao"
Học viên KHÔNG cần là dev. Bạn cần biết bấm "Hỏi AI" đúng lúc.

Tại sao có 4 sub-step nhỏ?
Facebook cố tình thiết kế phức tạp — vì token có quyền đăng bài thay bạn, không thể cho dễ. Bạn phải "đổi" token 3 lần: ngắn → dài → vĩnh viễn. Mỗi lần đổi là 1 lệnh đơn giản, nhưng phải làm đúng thứ tự.
3.1 — Lấy Short-lived User Token (sống 1 giờ)
Vào Graph API Explorer: https://developers.facebook.com/tools/explorer
Góc phải có dropdown Meta App → chọn app bạn vừa tạo (dh-content-bot).
Phía dưới có nút Add a Permission — thêm 4 permissions sau (gõ tên trong ô tìm rồi tick):
pages_manage_posts
pages_read_engagement
pages_show_list
public_profile
Bấm Generate Access Token (nút xanh dương phía trên).
Popup hiện ra → Continue as [Tên bạn] → chọn Page bạn muốn đăng (chỉ tick Page kinh doanh, không tick tất cả) → Continue → tick tất cả permissions → Save.
Bạn được trả về Graph Explorer, ô Access Token bây giờ có 1 chuỗi dài bắt đầu bằng EAA....
Kiểm tra điểm
Bạn copy được 1 chuỗi token bắt đầu bằng EAA, dài ~200 ký tự. Token này chỉ sống 1 giờ — phải đổi thành long-lived ngay ở 3.2.

3.2 — Đổi sang Long-lived User Token (sống 60 ngày)
AI-First — đừng tự gõ terminal
Từ đây tới hết Bước 03, bạn sẽ thấy nhiều lệnh curl. Không cần biết curl là gì. Mở agent coding (Cursor/Antigravity/Claude Code), paste lệnh + bảo: "Chạy lệnh này giúp tôi, giải thích output trả về." Agent sẽ chạy và dịch kết quả cho bạn.

Cách này cũng an toàn hơn — agent biết thay placeholder thế nào, biết debug nếu lỗi, biết gợi ý nếu bạn paste sai chỗ.

Mở terminal trên máy (Terminal trên Mac, PowerShell/CMD trên Windows), paste lệnh sau — thay 3 chỗ in đậm bằng giá trị thật của bạn:

curl -G "https://graph.facebook.com/v18.0/oauth/access_token" \
  -d "grant_type=fb_exchange_token" \
  -d "client_id=APP_ID_CỦA_BẠN" \
  -d "client_secret=APP_SECRET_CỦA_BẠN" \
  -d "fb_exchange_token=SHORT_LIVED_TOKEN_TỪ_3.1"
Trả về JSON kiểu:

Output mong đợi
{"access_token":"EAAxxxxxx...","token_type":"bearer","expires_in":5183944}
Copy chuỗi access_token này — đây là Long-lived User Token sống 60 ngày.

3.3 — Đổi sang Permanent Page Token (KHÔNG hết hạn)
Đây là token cuối cùng — vĩnh viễn. Cần biết User ID của bạn trên Facebook trước.

Lấy User ID:
curl "https://graph.facebook.com/v18.0/me?access_token=LONG_LIVED_TOKEN_TỪ_3.2"
Output
{"name":"Tên Bạn","id":"10223611603830114"}
Copy id — đây là User ID (chuỗi số dài).

Lấy Permanent Page Token:
curl "https://graph.facebook.com/v18.0/USER_ID/accounts?access_token=LONG_LIVED_TOKEN"
Output
{
  "data": [
    {
      "access_token": "EAAxxxxxxx...",  ← ĐÂY là Permanent Page Token
      "category": "Brand",
      "name": "Tên Page của bạn",
      "id": "123456789012345"          ← ĐÂY là Page ID
    }
  ]
}
Copy 2 chuỗi:

access_token — đây là Permanent Page Token. KHÔNG bao giờ hết hạn.
id — đây là Page ID của bạn. Cũng cần lưu.
3.4 — Kiểm tra token vĩnh viễn
Để chắc chắn token thật sự không hết hạn, vào Access Token Debugger: https://developers.facebook.com/tools/debug/accesstoken/

Paste Permanent Page Token vào ô.
Bấm Debug.
Tìm dòng Expires — phải là Never.
Kiểm tra điểm — quan trọng nhất
Nếu thấy Expires: Never — bạn đã thành công. Token này lưu vào file .env là dùng vĩnh viễn.

Nếu thấy thời gian hết hạn — bạn đã làm sai 1 bước. Quay lại 3.2 hoặc 3.3, làm lại cẩn thận.

⚠️ Troubleshoot — Bước 03
"Token short-lived ở 3.1 không có chuỗi EAA..."
"3.2 trả về error 'Invalid OAuth access token'"
"3.3 trả về data trống []"
"Permanent token vẫn hiện expires"
"Tôi không quen terminal, làm sao?"

BƯỚC 05
Vibe code skill bằng agent coding
Mở agent coding (Cursor / Antigravity / Claude Code) trong thư mục my-skills/ bạn đã tạo ở Ngày 16. Paste prompt:

Prompt · Vibe skill tao-creative-fb (AI-First)

Tôi cần tạo 1 Claude Skill Cấp 3 tên là "tao-creative-fb" — sản xuất full content (ảnh + văn bản) cho Facebook business page của tôi.

QUAN TRỌNG: Mỗi output phải gồm CẢ ẢNH VÀ VĂN BẢN — không phải chỉ ảnh. Một bài đăng FB hoàn chỉnh = ảnh + caption. Một creative ads hoàn chỉnh = ảnh ads + ad copy.

Skill này có 2 mode:

MODE 1 — CONTENT FREE (auto-post Page hằng ngày):
- Trigger: user nói "tạo content cho ngày mai", "gen bài Page", "content free", "content organic"
- Behavior: 
  + Bước A: Gen 3 ý tưởng (mỗi ý: tiêu đề + angle ngắn) → gửi cho user chọn
  + Bước B: User chọn 1 ý → agent gen FULL CONTENT cho ý đó:
      * 1 ảnh đẹp 1024x1024 bằng GPT Image API (model gpt-image-1, quality low cho rẻ)
      * 1 caption hoàn chỉnh ~80-150 từ (hook + body + soft CTA) theo brand voice
  + Bước C: Show preview ảnh + caption cho user duyệt
  + Bước D: User OK → POST cả ảnh + caption lên Facebook Page qua Graph API endpoint /{page-id}/photos (param: url=image_url, caption=caption_text)

MODE 2 — CREATIVE ADS (gen thủ công, không auto đăng):
- Trigger: user nói "tạo creative ads", "gen ads", "cần creative cho chiến dịch"
- Behavior: gen 3 BỘ creative — mỗi bộ là 1 cặp ghép đôi:
